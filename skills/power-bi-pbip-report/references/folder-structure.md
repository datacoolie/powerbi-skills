# PBIR Report Folder Structure

Complete annotated folder tree for a `.Report/` generated in PBIR format.

## Legend
- **[REQUIRED]** — Must exist for valid PBIR report
- **[OPTIONAL]** — Created as needed
- Schema URLs are for the JSON `$schema` property

## Folder Tree

```
<ProjectName>.Report/
│
├── definition.pbir                           [REQUIRED]
│   Schema: definitionProperties/2.0.0/schema.json
│   Links the report to its semantic model (by relative path or connection).
│
├── .platform                                 [OPTIONAL - auto-generated]
│   Git integration metadata (JSON). Contains config.version, metadata.type,
│   metadata.displayName. Typically auto-generated by Power BI service.
│
├── definition/                               [REQUIRED]
│   │
│   ├── report.json                           [REQUIRED]
│   │   Schema: report/3.2.0/schema.json
│   │   Report-level settings: theme collection, settings, resource packages.
│   │
│   ├── reportExtensions.json                 [OPTIONAL]
│   │   Schema: reportExtension/1.0.0/schema.json
│   │   Report-level measures and extensions.
│   │
│   ├── version.json                          [REQUIRED]
│   │   Schema: versionMetadata/1.0.0/schema.json
│   │   PBIR version metadata. Currently "2.0.0".
│   │
│   ├── pages/                                [REQUIRED]
│   │   │
│   │   ├── pages.json                        [REQUIRED]
│   │   │   Schema: pagesMetadata/1.0.0/schema.json
│   │   │   Declares page order (pageOrder array) and active page (activePageName).
│   │   │   Each entry in pageOrder must match a page folder name.
│   │   │
│   │   ├── <page-slug>/                      [one per page]
│   │   │   │   Folder name = slugified displayName (e.g., "sales-overview")
│   │   │   │   Must match the "name" property inside page.json.
│   │   │   │
│   │   │   ├── page.json                     [REQUIRED per page]
│   │   │   │   Schema: page/2.1.0/schema.json
│   │   │   │   Page properties: name, displayName, displayOption, dimensions,
│   │   │   │   type (Tooltip/Drillthrough), visibility, interactions, filters.
│   │   │   │
│   │   │   └── visuals/                      [OPTIONAL - only if page has visuals]
│   │   │       │
│   │   │       ├── <visual-slug>/            [one per visual]
│   │   │       │   │   Folder name = type-prefixed description (e.g., "card-total-revenue")
│   │   │       │   │   Must match the "name" property inside visual.json.
│   │   │       │   │
│   │   │       │   ├── visual.json           [REQUIRED per visual]
│   │   │       │   │   Schema: visualContainer/2.7.0/schema.json
│   │   │       │   │   Visual definition: type, position, query, formatting, filters.
│   │   │       │   │
│   │   │       │   └── mobile.json           [OPTIONAL per visual]
│   │   │       │       Schema: visualContainerMobileState/2.3.0/schema.json
│   │   │       │       Mobile layout override: position & formatting on phone canvas.
│   │   │       │
│   │   │       └── ...                       [additional visual folders]
│   │   │
│   │   └── ...                               [additional page folders]
│   │
│   └── bookmarks/                            [OPTIONAL]
│       │
│       ├── bookmarks.json                    [REQUIRED if bookmarks exist]
│       │   Schema: bookmarksMetadata/1.0.0/schema.json
│       │   Bookmark groups and ordering.
│       │
│       └── <bookmark-slug>.bookmark.json     [one per bookmark]
│           Schema: bookmark/2.1.0/schema.json
│           Bookmark state: affected visuals, filters, spotlight.
│
├── StaticResources/                          [OPTIONAL]
│   │
│   ├── RegisteredResources/                  [OPTIONAL]
│   │   Custom resources registered with the report:
│   │   - Custom theme JSON files (*.json)
│   │   - Embedded images (*.png, *.jpg, *.svg)
│   │   These are referenced by report.json → resourcePackages.
│   │
│   └── SharedResources/                      [OPTIONAL]
│       └── BaseThemes/
│           └── CY24SU06.json                 [base theme file]
│               The default Power BI base theme. File name varies by release.
│               Referenced by report.json → themeCollection → baseTheme.
│
└── <ProjectName>.Report.json                 [OPTIONAL - auto-generated]
    Local settings file. Not part of the PBIR definition.
```

## Cross-Reference Rules

| Source | Property | Must Match |
|---|---|---|
| Page folder name | — | `page.json` → `name` |
| Visual folder name | — | `visual.json` → `name` |
| `pages.json` → `pageOrder[i]` | Each string | A page folder name |
| `pages.json` → `activePageName` | String | A page folder name |
| `page.json` → `visualInteractions.source` | String | A visual folder name on that page |
| `page.json` → `visualInteractions.target` | String | A visual folder name on that page |
| `report.json` → `resourcePackages[].items[].path` | String | File in `RegisteredResources/` |
| `definition.pbir` → `datasetReference.byPath.path` | Relative path | `.SemanticModel/` sibling folder |

## Minimum Viable Report

The absolute minimum files for a valid PBIR report:

```
MyReport.Report/
├── definition.pbir
└── definition/
    ├── report.json
    ├── version.json
    └── pages/
        ├── pages.json
        └── overview/
            └── page.json
```
