## Slot specifications

| Slot | x | y | w | h | Visual type | Notes |
|---|---|---|---|---|---|---|
| Title band | 0 | 0 | 1664 | 52 | textbox + slicer(Period) | Title + period |
| Category block Г— 8 | 0 | 62 | 1664 | 874 | 4Г—2 grid of category blocks | Each block = label header + 8-12 small KPI card / shape visuals. Typical block size ~316w Г— 332h |

Gutters: 16px between primary zones; 8px inside KPI card rows.

---

## Navigation

- Reachable from the report's top-nav chiclet strip or landing page. Include a small 'Home' actionButton in the header when not the landing page.
- Cross-links out to related drillthrough / detail pages should be surfaced via card-level actions, not a separate nav rail.

---

## Theme + iconography guidance

- **Palette:** Neutral background; each block header uses a faint background tint. KPI traffic-light (green/amber/red) indicators reserved for threshold breaches.
- **Logo:** Title band top-left at (16, 8) max height 20px.
- **Icons:** Category glyph next to each block header (max 16px).
- **Fonts:** Title 14pt, block header 10pt Bold, card value 12pt, label 7pt. Micro-type is intentional вҖ” audience is domain-expert.

---

## When NOT to use this layout

- вқҢ Audience is exec / casual вҖ” information overload. Use `scorecard-kpi-grid` (12 KPIs) instead
- вқҢ Any KPI is business-critical on its own вҖ” promote it to its own page with context
- вқҢ Mobile or small display (< 24" monitor) вҖ” layout becomes unreadable

---

## Customization allowed

- Change block layout to 3Г—3 if category count is 9
- Colour block headers by domain (Finance / Ops / Customer)
- Link each KPI card to a drillthrough for that metric

## Customization NOT allowed

- Adding chart visuals вҖ” this is a card/shape page by design. Charts go on sibling pages
- Reducing visual count below ~40 вҖ” use `scorecard-kpi-grid` instead
- Removing category labels вҖ” the layout depends on block structure for legibility
